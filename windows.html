The Common Lisp Cookbook - Setting up an IDE with Emacs on Windows

Emacs is the preferred Lisp source code editor for most CL developers; however, it is not easy for an Emacs neophyte to get an Emacs environment set up and configured properly for Lisp development. The problem is compounded if the development platform is Microsoft Windows since Unix (and variants) is the more common platform used by experienced Lisp developers. In order to help a Lisp beginner set up an Emacs Common Lisp development environment on a Microsoft Windows PC, this chapter will lay out step-by-step instructions for installing all the necessary components for each of the most popular Common Lisp implementations on Windows:

  • CLISP: ILISP will be used in Emacs to provide the development environment
  • Allegro Common Lisp (ACL): Franz's own "ELI" will be used in Emacs
  • Corman Common Lisp: The Emacs eshell will be used - although this is a simplistic approach, it is more usable than the raw clconsole application provided with this implementation
  • Xanalys LispWorks: ILISP will be used in Emacs to provide the development environment TO-DO (Note: the trial version of the product doesn't provide a non-gui Lisp image and this is necessary if the product is to be used with Emacs. Xanalys will be providing me with an evaluation version of LispWorks so that I can provide an equivalent set of instructions for LispWorks in the future.)

By exactly following the installation instructions, the user will wind up with an Emacs setup that:

  • Allows the user to trial the most popular Lisp implementations for Windows
  • Allows the user to try out 3 different techniques (ILISP, ELI, eshell) for establishing a Lisp development environment in Emacs - each with its own sets of pluses and minuses
  • Configures Emacs so that it has keystroke bindings more similar to what a Windows user is accustomed to
  • Configures Emacs with a number of custom configuration settings that are commonly used

For each installation, an installation directory is suggested. It isn't necessary to install in that directory; however, if you use the suggested names, the provided Emacs start-up file (.emacs) will work "out-of-the-box". Please note the following:

  • In some cases, you will probably want to first unpack the files, retaining the directory structure, and then rename the high-level directory to the suggested name 
  • If you use alternative directory names, you will need to modify the .emacs file (instructions provided in step #7)
  • The instructions assume you are using Winzip; however, you can use any alternative utility
  • When you unpack .gz files with Winzip, it will ask you "Should WinZip decompress it to a temporary folder and open it?" - click on "No"

1. Installing Emacs

  • Download the precompiled Emacs binaries for Windows from ftp://ftp.gnu.org/gnu/windows/emacs/21.1/emacs-21.1-fullbin-i386.tar.gz
  • Use Winzip to unpack the files (Suggested directory: c:\emacs)
  • Using Run on the Windows Start menu, enter "c:\emacs\bin\runemacs.exe" to verify that the program works. On the Help menu, there is an Emacs on-line tutorial. If you are unfamiliar with Emacs, you should (at least) complete this tutorial before proceeding any further. The remainder of this chapter's instructions will assume that you have at least basic familiarity with the use of Emacs.
  • Create the following directories (used later): 

c:\home\
     c:\home\site\
     c:\home\docs\
     c:\home\lisp\

2. Installing CLISP

  • Download the precompiled CLISP binaries for Windows from http://prdownloads.sourceforge.net/clisp/clisp-2.27-win32.zip
  • Use Winzip to unpack the files (Suggested directory: c:\clisp-2.27)
  • The install.bat file for CLISP runs the install.lisp program; however, the version of the program included with the CLISP 2.27 Windows distribution will not work properly with some versions of Windows. It is necessary to download the CVS version of the file. In addition, the batch file that is generated by the install.lisp program will not work properly if CLISP has been installed in a directory that has spaces in it's name (e.g. -- "C:\Program Files\"). Therefore, you should install CLISP either in the suggested directory or one that doesn't have spaces in it's name.
  • If you don't have a CVS utility, you will need to download one in order to get the CVS version of install.lisp (Note: You will need CVS later on as well):
    • Download CVS from http://ftp.cvshome.org/win32/cvs1-11-1p1.zip
    • Use Winzip to unpack the cvs.exe file into the c:\home\site directory or into a directory that is in your PATH.
  • Download the install.lisp utility from CVS and put it into the c:\clisp-2.27\src directory, replacing the version of install.lisp that is already there. To do this, enter the following commands from a dos command line (Note: when you are prompted for the CVS password after entering line#2, just press enter):

    cd c:\home\site
    cvs -d:pserver:anonymous@cvs.clisp.sourceforge.net:/cvsroot/clisp login
    cvs -z3 -d:pserver:anonymous@cvs.clisp.sourceforge.net:/cvsroot/clisp co clisp/src/install.lisp
    cd c:\clisp-2.27\src\
    copy c:\home\site\clisp\src\install.lisp install.lisp

    This will result in the install.lisp file replacing the version of install.lisp that is already in the c:\clisp-2.27\src\ directory. There will still be the (mostly empty) downloaded CVS clisp directory tree in the c:\home\site directory - this can be deleted now.

  • You now need to complete the installation process for CLISP by running the install.bat file. To do this, go to a dos command line, cd to the c:\clisp-2.27 directory and run the install.bat file. This will result in the creation of a clisp.bat file on the desktop. Move this file from the desktop to the c:\home\lisp directory. The clisp.bat file will be used to start CLISP from within Emacs. (Note: Some people choose to edit this batch file to include other parameters. I personally add the "-ansi -I -q" switches to mine. This makes CLISP more ILISP friendly, ANSI CL compliant and eliminates the startup banner. Once you have moved the clisp.bat file to the c:\home\lisp directory, you may want to modify this file - an explanation of all of the different switch settings is in the c:\clisp-2.27\doc\clisp.html file).
  • The clisp-indent.el utility provides better Lisp indentation - it is actually part of CLISP but isn't included in the Windows zip file. You need to download it separately from CVS and put it into the c:\home\site directory. To get it from CVS, enter the following commands from a dos command line (Note: when you are prompted for the CVS password after entering line#2, just press enter):

    cd c:\home\site
    cvs -d:pserver:anonymous@cvs.clisp.sourceforge.net:/cvsroot/clisp login
    cvs -z3 -d:pserver:anonymous@cvs.clisp.sourceforge.net:/cvsroot/clisp co clisp/emacs/clisp-indent.el
    copy clisp\emacs\clisp-indent.el  clisp-indent.el

    This will result in the clisp-indent.el file being in the c:\home\site directory. There will still be the (mostly empty) downloaded CVS clisp directory tree in the c:\home\site directory - this can be deleted now.

  • Create a file named _clisprc.lisp in the c:\home directory. This file can have any initializations that need to occur when CLISP starts. For the time being, just leave it empty.

3. Installing ACL

  • Download the trial version of Franz's ACL for Windows from http://www.franz.com/downloads/index.lhtml#acl  
  • (Note: you will need to agree to their license and provide an email address. Franz will send you an email with instructions on how to download the temporary license key. The key works for 60 days and a new one can be downloaded before that one expires)
  • Run the downloaded installation program (acl61_trial.exe) and install ACL (Suggested directory: c:\Program Files\acl)
  • Install the license key as per the instructions that you received from Franz
  • From the Windows Start menu, navigate to the Allegro CL group and startup ACL
  • On the ACL Tools menu, select the Options menu item. Click on the Editor tab and select the checkbox titled "Open files in GNU Emacs". Click Ok and exit ACL.
  • The fi-site-init.el file that is in the ACL eli directory and is used to startup ACL from Emacs will not work properly in Emacs v21.1. With the current version of ACL, it is necessary to make the highlighted (in red) change below to the code in the file in order for it to work in Emacs v21.1:

(setq fi::emacs-type
  (let ((case-fold-search t))
    (cond ((or (string-match "xemacs" emacs-version)
                     (string-match "lucid" emacs-version))
               (cond ((string-match "^2[01]\." emacs-version) 'xemacs20)
                         (t 'xemacs19)))
              ((string-match "^2[01]\." emacs-version) 'emacs20)
              ((string-match "^19\." emacs-version) 'emacs19)
              ((string-match "^18\." emacs-version) 'emacs18)
              ((boundp 'epoch::version) 'epoch)
              (t
               (error
                 "%s is not supported by this version of the emacs-lisp interface."
                 emacs-version)))))

4. Installing Corman CL

  • Download the precompiled Emacs binaries for Corman Lisp for Windows from http://corman.net/CormanLisp/CormanLisp_1_5_Setupex.exe
  • Run the downloaded installation program (CormanLisp_1_5_Setupex.exe) and install Corman CL (Suggested directory: C:\Program Files\Corman Tools\Corman Lisp 1.5)
  • Create a batch file to use for starting Corman from within Emacs. The file should be called corman.bat and it should be created in the c:\home\lisp directory. It should have the following lines in it (Note: Make certain that the quote signs are exactly as shown here):

@echo off
"C:\Program Files\Corman Tools\Corman Lisp 1.5\clconsole.exe" -image "C:\Program Files\Corman Tools\Corman Lisp 1.5\cormanlisp.img"

5. Installing additional Emacs utilities

  • The current 5.11 version of the ILISP package (Lisp development environment used with clisp) does not work properly with Emacs v21.1 (also, the latest version of the Hyperspec needs the CVS version as well); therefore, it is necessary to download the CVS version of this package.
  • Go to a dos command line and enter the following commands to download the CVS version of ILISP (Note: when you are prompted for the CVS password after entering line#2, just press enter):

cd c:\home\site
cvs -d:pserver:anonymous@cvs.ILISP.sourceforge.net:/cvsroot/ILISP login 
cvs -z3 -d:pserver:anonymous@cvs.ILISP.sourceforge.net:/cvsroot/ILISP co ILISP

There should now be an ILISP directory in the c:\home\site directory.

  • Download the buildilisp.bat file from here (Note: if you used the recommended directory names for Emacs and ILISP, you don't need to make any changes to this batch file - otherwise, edit the file as necessary). It doesn't matter where you place this file.
  • From a command line, run the buildilisp.bat file
  • Download the w3 package (needed for browsing the Lisp documentation) from ftp://ftp.xemacs.org/pub/xemacs/emacs-w3/w3.tar.gz 
  • Use Winzip to unpack the files (Suggested directory: c:\home\site\w3)
  • From a command line, cd to the c:\home\site\w3  directory and run the build.bat command (Note: if you unpacked Emacs into a directory other than c:\emacs, you will need to modify this batch file to point to the correct directory location)
  • Download the cua.el utility (provides Windows standard key mappings)  from http://www.cua.dk/cua.el and put it into the c:\home\site directory
  • Download the sample .emacs file from here and place it in the c:\home directory

6. Installing on-line documentation

  • Download the Common Lisp Hyperspec reference from http://www.xanalys.com/software_tools/reference/HyperSpec/HyperSpec-6-0.tar.gz
  • Use Winzip to unpack the files (Suggested directory: c:\home\docs)
  • Download the Common Lisp the Language reference from http://www-2.cs.cmu.edu/afs/cs.cmu.edu/project/ai-repository/ai/lang/lisp/doc/cltl/cltl_ht.tgz
  • Use Winzip to unpack the files (Suggested directory: c:\home\docs)

7. Configuration steps

  • Setup environmental variables either in the autoexec.bat file (e.g. - Windows/98) or the Control Panel/System (e.g. - Windows/NT/2000)
  • HOME: Should be set to c:/home (Note: use "/" rather than "\")
  • PATH: The c:\emacs\bin and c:\home\lisp directory paths should be prepended to the existing PATH to ensure that the Emacs executable and the CLISP executables can be found (e.g. -- set path=c:\emacs\bin;c:\home\lisp;%path%)
  • If the specified directory locations were used, no changes to the .emacs file should be necessary. If file locations were changed, the directory locations that are specified in the first section of the .emacs file (the "Site-Specific Variables" section) should be modified with the correct directory locations (Note: when specifying directory locations in the .emacs file, use "/" rather than "\" as the directory separator character).
  • Reboot the PC after making the above changes and configuration settings
  • Start Emacs (either by using the Start menu's Gnu Emacs shortcut or by using Run on the Windows Start menu with the command "c:\emacs\bin\runemacs.exe") and do the following:
    • Compile the .emacs file by pressing "B" when the cursor is on the file in the initial dired window (answer "y" to the "Byte-Compile .emacs (y or n)" prompt)
    • Double-click on the "site" directory to drill down into that directory
    • Compile the clisp-indent.el and cua.el files in the same manner as was done for .emacs
    • Start CLISP by pressing F5
    • After starting ILISP/CLISP in Emacs, use "M-x ilisp-compile-inits" to compile the CLISP Lisp initialization files
  • All configuration steps should now be complete - exit Emacs by pressing "M-F4"

8. Testing the environments

  • Start Emacs (either by using the Start menu's Gnu Emacs shortcut or by using Run on the Windows Start menu with the command "c:\emacs\bin\runemacs.exe") and do the following:
    • Test that CLISP runs by pressing "F5" (CLISP should open in the bottom window and the ILISP menu should appear in the Emacs menu bar)
      • Click in the *clisp-hs* window and press "C-h b" (this will show you the keybindings that are in effect for ILISP/CLISP)
      • Click in the *Help* window and scroll through the keybinding list. When finished, press "F4" to close that window
      • Click in the *clisp-hs* window and press "F4" to close that window
    • Click in the dired window and press "M-C-F5" (the minibuffer window at the bottom will say "lisp-used: :acl" indicating that you have toggled to ACL mode)
    • Test that ACL runs by pressing "F5"
      • The ACL application should start up, click on the ok button after the splash screen
      • In the ACL application, select the menu option File/Open and open the ftp.cl file in the examples directory of ACL (it should open in Emacs and the ACL menus should appear in the Emacs menu bar.
      • Click in the ftp.cl Emacs window and press "C-h b" (this will show you the keybindings that are in effect for ELI/ACL)
      • Click in the *Help* window and scroll through the keybinding list. When finished, press "F4" to close that window
      • Click in the ftp.cl Emacs window and select the ACLFile/Exit Allegro CL menu option (this will close down the ACL connection and the ACL application will end)
    • Click in the dired window and press "M-C-F5" (the minibuffer window at the bottom will say "lisp-used: :corman" indicating that you have toggled to Corman mode)
    • Test that Corman runs by pressing "F5"
      • The Emacs eshell should start up and it should start up Corman CL (the Corman CL prompt will be a "?")
      • Click in the *eshell* window and press "C-h b" (this will show you the keybindings that are in effect for eshell/Corman)
      • Click in the *Help* window and scroll through the keybinding list. When finished, press "F4" to close that window
      • Click in the *eshell* window and press "F4" to close that window
    • Exit Emacs by pressing "M-F4"

9. Sample .emacs file

  • Following is a brief explanation of each of the sections of the sample .emacs file:
    • Site-Specific Variables: Contains directory locations for the different Lisp implementations, Lisp documentation and Emacs-specific files. It also sets the default Lisp implementation to CLISP.
    • Initial Code Load: Loads Emacs files that are needed later on
    • System Customizations: Some standard customizations that make Emacs easier to use.
    • Programming - Common Lisp: CL-specific setup
    • Programming -  Elisp: Emacs Lisp-specific setup
    • Standard Key Overrides: Some convenience key setups
    • Lisp Key Overrides: ILISP & ELI have their own key mappings - here are some extra ones
    • Windows Key Overrides: These settings make Emacs feel more like a Windows application
    • Start Directory: After Emacs starts up, the start directory will be the HOME directory
